Personal Blog API (FLASK)

Introduction

Welcome, this Blog API contains 30 routes with the exeption of this route that displays this page. I will show you all the routes with their given request methods and parameters if needed. Without further delay, let's delve in.

How To Use

For experienced python and github users, you can clone the repository @ Githib link. Note: You will have to make slight chances to the database_uri to be able to use sqlite3 for internal database storage, create your own .env file to store sensitive info like jwt secret_key, flask secret_key(s); BOOOOM!!! You know your way from there.

For those without python background and may or maynot have an experience with github; you are the reason i made this welcome page. All you have to do is follow the information on the table below; any further question or issues, reach me via my email below or open and issue in my git repository Git repo. Thank you.

All devops/users note: At the end of the the base url, always add /v1else you risk not accesing the site.

Note:All routes requires you to be logged in. It uses JWT tokens for authentication and authorization.
All responses are strictly in JSON.

Route Method Paramters Usage/Description
https://www.johnstares.pythonanywhere.com/v1 GET None BASE URL
Notice the /v1 at the end of the .com? it is required to access this site else you get page not found. The v1 stands for the version of the blog.
Any other routes falls behind the base url eg. https://www.johnstares.pythonanywhere.com/v1/blogs
This page you are on, is the BASE URL.
/register POST None To add a user to the blog.
This route content-type is multipart/form-data just incase the user wants to add a profile image during registration.
eg. key=data value={"username": "John Stares", "email": "johnstares@gmail.com", "password": "yourpassword"}
alongside an image eg. key=image value=name of image.PNG
/login POST None This route content-type is application/json
{ "username": "John Stares", "password": "yourpassword" }
/blogs POST None To make a blog post. content-type is multipart/from-data
key is data, value is title, content, category, tag, image_name;
key is image for attachment of images together with the blog.
eg. key=data, value={"title": "The legend king", "content": "your blog content": "category": "Folk-Tale", "tag": ["epic", "king", "legendary"], "image_name": "picture of the king being celebrated."}
alongside the image key=image value=name of pciture.PNG
/blogs GET None To fetch all post. This routes brings all post from the database if working directly with this site. post made and not deleted by other users including yours will be shown here.
/delete-blog DELETE /1 Use to delete a blog.
The parameter /1 is the blog ID.
/delete-tag DELETE /1 Use to delete a tag.
The parameter /1 is the tag ID.
/view-users GET None To get all the users in the database.
Doesn't require you to be logged in but it uses a rate-limiter and only 3 request per day.
/add-comment POST /1 To add a comment to a blog.
The /1 is the ID of the blog you want the comment to go to.
content-type is application/json
{ "content": "your comment". }
/view-comment GET None To get all the comments on every blog made not only by you.
delete-comment DELETE /1 To delete a comment
The /1 is the ID of the comment you want to delete.
/reply-comment POST /1/1 To reply a comment
The first /1 is the comment ID and the second /1 is the blog ID.
content-type is application/json
{ "reply": "your reply" }
/reply-reply POST /1/1/1 To reply a reply.
The first /1 is the comment ID, the second /1 is the blog ID and the third /1 is the reply ID.
content-type is application/json
{ "reply": "your reply" }
/delete-reply DELETE /1 To delete a reply.
The /1 is the ID of the reply.
/view-reply GET None To get all replys made to a comment even those not made by you on your own blog.
/view-replies GET None To get all replies made to a reply made to a comment even those not made by you on youw blog.
/like/blog/id POST /1 To make a like to a blog.
The /1 is the ID of the blog you want to like.
/like/comment/id POST /1 To make a like a comment.
The /1 is the ID of the comment you want to like.
/like/reply/id POST /1 To make a like to a reply.
The /1 is the ID of the reply you want to like.
/delete-likes/id DELETE /1 Use to unlike a blog or a comment or a reply.
/search GET None To search for a blog either by author, category, tag, title or date.
It uses a search parameter (a for author, c for category, q for title, p for date and t for tag).
You can make a specific search using author, category and title at the same time.
Doesn't require you to sign in but uses a rate-limiter of 3 request per minute.
/serve-images/filename GET /filename A route that works behind the scene in ensuring profile pictures and blog images are delivered.
user-profile/1 GET /1 (optional) To get details and blogs post made by a user.
If it carries a parameter /1 it means you are searching for another users profile but if it doesn't it means, you are entering your own profile.
/search-author GET None To search for a user.
This route uses a search parameter a and returns an ID of ther user if found; you will now user the ID on /user-profile/id.
/password-update PATCH None To change password.
content-type is application/json
{ "new password": "your new password" } a rate-limiter is used to limit password update twice in a month.
/username-update PATCH None To change password.
content-type is application/json
{ "username": "your new username" }
/update-profile-pic PATCH None To update profile picture.
content-type is multipart/form-data.
key=image value=profilepic.PNG
/delete-account DELETE None To delete ones account. Best to be done after testing.
/follow/id POST /id To follow a user
/logout POST None To logout of ones account.
